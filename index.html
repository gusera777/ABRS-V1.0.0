⚙ Pengaturan ABRS

Modal & Data Feed

Disimpan lokal di browser (localStorage), tidak dikirim kemanapun selain api.twelvedata.com.

Basket Hedging + Reverse Martingale

Lot entry pertama. Tabel Lot Level 2-10 dihitung otomatis secara progresif (x3 lalu x2 tiap level) dari nilai ini.
Jarak antar level breakout/pending order, dalam point.
Sesuaikan per instrumen — XAU/USD ≈ 0.01, BTC/USD ≈ 1, US30/USD ≈ 1, EUR/USD atau GBP/USD ≈ 0.0001.
Semua posisi ditutup (Close All) begitu total floating profit basket mencapai persentase ini dari equity awal cycle. Contoh: 0.01 = 1%.

Kunci Profit / Breakeven+ (Opsional)

Begitu profit berjalan basket mencapai nilai ARM di bawah, kunci diaktifkan. Jika profit lalu turun kembali sampai ke level BE+, seluruh posisi otomatis ditutup (Close All) untuk mengamankan profit yang sudah didapat, lalu cycle mulai lagi dari Level 1.
Contoh: 50 → kunci diaktifkan begitu profit basket berjalan mencapai $50.
Contoh: 30 → setelah ARM, jika profit turun kembali ke $30, Close All otomatis dieksekusi mengamankan profit $30 tsb.

Proteksi Tambahan (Jaring Pengaman, Opsional)

Sistem basket martingale tidak punya batas rugi bawaan. Jaring pengaman ini di luar dokumen rules asli, disarankan tetap aktif untuk akun riil.
Cycle ditutup 'LOSS' lebih awal di level ini, sebelum menyentuh 100% Global Risk (EMERGENCY).

Headless Runner (jalan tanpa app dibuka)

Aplikasi ini murni client-side — begitu tab/browser ditutup, semua proses berhenti. Untuk sistem yang tetap trading walau app tidak dibuka, jalankan node headless-runner.js di komputer/server yang tetap menyala (lihat README). Riwayat cycle & event yang dihasilkannya tersimpan di file headless-store.json — impor file itu di sini untuk digabung ke dashboard ini kapan saja.
